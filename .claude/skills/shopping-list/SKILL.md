# 買い物リスト生成

選択したレシピの材料を集計し、買い物リストとして表示します。

## 参照ファイル

- `recipes/items/*.md` - 登録済みレシピ
- `recipes/ingredients.csv` - 材料マスター

## 手順

### 1. レシピ一覧を読み込む

`recipes/items/` 内の全MDファイルを読み込み、レシピ名・カテゴリ・栄養素を把握する。

### 2. 作りたいレシピを選択

AskUserQuestion（multiSelect: true）でレシピを選択させる。
- 登録済みレシピを選択肢として表示（カテゴリ・kcal付き）
- 複数選択可能

### 3. 人数・倍量の確認

選択されたレシピごとに AskUserQuestion で質問:
- 「{レシピ名}（{N}人前）を何セット作りますか？」
- 選択肢: 1セット / 2セット / 3セット / Other
- 1セットならそのまま、2セットなら材料を2倍にする

### 4. 材料を集計

全レシピの材料を合算する。同じ材料は分量を合計する。

**合算ルール:**
- 同じ単位（g, ml, 個など）の材料はそのまま加算
- 「大さじ」「小さじ」は本数のまま加算（例: 大さじ3 + 大さじ5 = 大さじ8）
- 「少々」「適量」「お好み」は合算せず、そのまま表記
- 「パック」「株」「本」などは個数を加算

### 5. 調味料の確認

集計した材料を「食材」と「調味料」に分類する。

分類基準（ingredients.csv の category を参照）:
- **食材**: 肉類、魚介類、野菜、果物、穀類、乳製品、その他
- **調味料**: 調味料カテゴリのもの（醤油、みりん、料理酒、砂糖、塩、味の素、だし等）

AskUserQuestion（multiSelect: true）で調味料リストを表示:
- 「以下の調味料で買い足しが必要なものを選んでください（家にあるものはスキップ）」
- 集計された調味料を選択肢として表示
- 「全部必要」「全部不要」の選択肢も含める

### 6. 買い物リストを表示

以下のフォーマットで出力:

```
## 🛒 買い物リスト

### 選択レシピ
- {レシピ名1} ×{N}セット
- {レシピ名2} ×{N}セット

---

### 食材

| 材料 | 合計分量 |
|------|---------|
| {材料1} | {合計分量} |
| {材料2} | {合計分量} |

### 調味料（買い足し分）

| 調味料 | 合計分量 |
|--------|---------|
| {調味料1} | {合計分量} |
| {調味料2} | {合計分量} |

---

### 栄養素合計（全レシピ・全セット分）

| kcal | タンパク質 | 脂質 | 炭水化物 |
|------|-----------|------|---------|
| {total_kcal} | {total_p}g | {total_f}g | {total_c}g |

（{合計人数}食分 → 1食あたり約{per_kcal}kcal）
```

手順6で表示した買い物リストをDiscord Webhookで送信する。

**Webhook URL:** `https://discord.com/api/webhooks/1482035017969434795/oy2xYGh1yXR2odt7SpppZ6lR7A8Fquls1A2t-0hMab78jRLaHshWyVZTNlUVuQTB5cZ-`

**送信方法:** Bash ツールで curl を使用:

```bash
curl -H "Content-Type: application/json" -X POST -d '{"content": "{買い物リストのテキスト}"}' "WEBHOOK_URL"
```

**送信フォーマット（Discord用・プレーンテキスト）:**

```
🛒 買い物リスト（{日付}）

📋 レシピ
・{レシピ名1} ×{N}セット
・{レシピ名2} ×{N}セット

🥩 食材
・{材料1} … {合計分量}
・{材料2} … {合計分量}

🧂 調味料（買い足し分）
・{調味料1} … {合計分量}
・{調味料2} … {合計分量}

📊 栄養素合計
{total_kcal}kcal / P:{total_p}g / F:{total_f}g / C:{total_c}g
（{合計人数}食分 → 1食あたり約{per_kcal}kcal）
```

**注意:**
- JSON文字列内のダブルクォートやバックスラッシュはエスケープすること
- 改行は `\n` で表現する
- Discord のメッセージ上限は2000文字。超える場合は食材と調味料を分割して2回送信する
- 調味料が「全部不要」の場合は調味料セクションを省略

**送信方法:** 手順7と同様に Bash ツールで curl を使用。買い物リストの送信と並列で実行する。

**送信フォーマット（Discord用・1レシピ1メッセージ）:**

```
🍳 {レシピ名}（{カテゴリ} / {N}人前 / {kcal}kcal）

📊 栄養素（1人前）
P:{protein}g / F:{fat}g / C:{carb}g

🥕 材料
・{材料1} … {分量}
・{材料2} … {分量}

📝 作り方
1. {手順1}
2. {手順2}
...
```

**注意:**
- 選択されたレシピごとに1メッセージずつ送信する（並列実行可）
- レシピMDファイルの「作り方」セクションから手順を抽出する（ポイントや補足は省略してコンパクトに）
- Discord のメッセージ上限は2000文字。超える場合は作り方を簡略化する
- セット数の情報はレシピ投稿には含めない（レシピ自体の情報のみ）

## 注意事項

- レシピが1件も登録されていない場合は「レシピが登録されていません。`/add-recipe` で登録してください」と案内する
- 調味料で「全部不要」が選ばれた場合、調味料セクションは非表示にする
- 同じ材料でも切り方の指示（斜め切り、5mm幅等）はリストに含めない（材料名のみ）
- 栄養素合計はセット数を反映した値を表示する
