# レシピ登録

対話形式で新しいレシピを登録し、`recipes/items/` にMDファイルとして保存します。

## 参照ファイル

- `CLAUDE.md` - レシピMDテンプレート
- `recipes/ingredients.csv` - 材料マスター（既存材料の参照用）
- `recipes/items/` - 既存レシピ（重複チェック用）

## 手順

### 1. 既存データを読み込む

- `recipes/ingredients.csv` を読み込み、登録済み材料を把握
- `recipes/items/` のファイル一覧を確認し、既存レシピ名を把握

### 2. レシピ情報を質問

AskUserQuestion で以下を順番に質問:

#### 質問1: 基本情報
- 「レシピ名を入力してください」（自由入力）

#### 質問2: カテゴリ
- 選択肢: 主菜 / 副菜 / 汁物 / 主食 / おやつ / ドリンク / Other

#### 質問3: 分量
- 「何人前ですか？」
- 選択肢: 1人前 / 2人前 / 3人前 / 4人前 / Other

#### 質問4: 材料
- 「材料と分量を入力してください（例: 鶏むね肉 200g, 塩 少々, ...）」
- 自由入力。カンマ区切りやリスト形式でOK
- 複数回に分けて追加も可能

#### 質問5: 作り方
- 「作り方を入力してください」
- 自由入力。番号付きリスト、または文章で記述

#### 質問6: タグ
- multiSelect: true
- 選択肢: 簡単 / 高タンパク / 低カロリー / 低脂質 / 作り置き / 時短 / ヘルシー / ガッツリ / Other
- スキップ可能

#### 質問7: メモ（任意）
- 「メモがあれば入力してください（スキップ可）」
- スキップ可能

### 3. 栄養素を計算

- 入力された材料から栄養素を推定
  - `ingredients.csv` に登録済みの材料は、そこから計算
  - 未登録の材料は一般的な栄養素データから推定
- 1人前あたりの kcal / タンパク質 / 脂質 / 炭水化物 を算出

### 4. 材料マスターを更新

- 入力された材料のうち `ingredients.csv` に未登録のものがあれば追加
- 推定した栄養素情報を含めて登録

### 5. レシピファイルを作成

`recipes/items/{レシピ名}.md` を以下のテンプレートで作成:

```markdown
# {レシピ名}

- カテゴリ: {カテゴリ}
- 分量: {N}人前
- タグ: {タグ1}, {タグ2}, ...

## 栄養素（1人前）

| kcal | タンパク質 | 脂質 | 炭水化物 |
|------|-----------|------|---------|
| {kcal} | {protein}g | {fat}g | {carb}g |

## 材料

| 材料 | 分量 |
|------|------|
| {材料1} | {分量1} |
| {材料2} | {分量2} |
| ... | ... |

## 作り方

1. {手順1}
2. {手順2}
...

## メモ

{メモ（あれば）}
```

### 6. 登録結果を表示

作成したレシピのサマリーを表示:

```
## レシピを登録しました

**{レシピ名}** ({カテゴリ})

| kcal | タンパク質 | 脂質 | 炭水化物 |
|------|-----------|------|---------|
| {kcal} | {protein}g | {fat}g | {carb}g |

材料: {材料数}品
ファイル: recipes/items/{レシピ名}.md
```

## 注意事項

- レシピ名が既存と重複する場合は上書きするか確認する
- 栄養素は推定値であることを明記する（正確な値が分かる場合はユーザー入力を優先）
- ファイル名に使えない文字（/ : など）はレシピ名から除去する
- 材料の分量表記は「少々」「適量」「大さじ1」など日本語の一般的な表記を許容する
